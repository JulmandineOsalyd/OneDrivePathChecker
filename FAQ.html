Fonctionnement de l'outil

En mode Synchronisation OneDrive, l'outil mesure la longueur du chemin Windows complet de chaque fichier, tel qu'il existe sur votre ordinateur. Par exemple :

C:\Users\julie\OneDrive - Mon Entreprise\Projets\2024\Document important.docx

Si ce chemin dépasse 255 caractères, Windows et OneDrive peuvent rencontrer des erreurs de synchronisation, des fichiers ignorés, ou des blocages lors de l'ouverture.

La limite de 255 caractères est une contrainte Windows qui s'applique au chemin complet depuis la racine du disque (C:\...).

En mode Migration SharePoint, l'outil simule l'URL relative que chaque fichier aura une fois migré sur SharePoint. Par exemple, si vous migrez vers le site RH et la bibliothèque Documents partagés, le fichier :

C:\Serveur\Archives\Contrats\2024\Contrat_Julie.docx

sera simulé comme :

/sites/RH/Documents partagés/Archives/Contrats/2024/Contrat_Julie.docx

C'est exactement le format qu'utilise Microsoft pour calculer la longueur du chemin SharePoint.

Un nom de site fictif est renseigné par défaut pour estimer les longueurs d'URL simulées. L'objectif est d'identifier les fichiers à risque — même avec un nom provisoire, les résultats restent exploitables pour prioriser vos renommages.

Vous pouvez également cocher la case « Je ne connais pas le site SharePoint cible » pour utiliser les valeurs par défaut et lancer l'analyse immédiatement.

Ces deux seuils correspondent à des niveaux de risque différents :

Limite Risque Contexte
255 caractères Avertissement Problèmes potentiels avec les applications Office, l'Explorateur Windows et certains outils tiers
400 caractères Critique Blocage total par SharePoint — migration impossible
Traitez en priorité les fichiers dépassant 400 caractères, puis ceux entre 255 et 400 pour garantir un usage fluide après migration.

Capacités et performance

L'outil est conçu pour analyser des arborescences de taille professionnelle :

Volume Performance
Jusqu'à 50 000 fichiers Rapide et fluide
50 000 à 100 000 fichiers Fonctionnel, quelques secondes d'attente
Au-delà de 100 000 fichiers Possible ralentissement selon la machine
Pour les arborescences très volumineuses, analysez les dossiers principaux séparément plutôt que l'intégralité du serveur en une seule fois.

Oui, l'outil peut analyser un lecteur réseau mappé (par exemple Z:\Partage\Services\RH). Il suffit de sélectionner le dossier via le bouton de sélection, comme pour n'importe quel dossier local.

Quelques points à garder en tête :

  • Performance : un lecteur réseau est plus lent qu'un SSD local — sur 20 000 fichiers, le scan peut prendre plusieurs dizaines de secondes selon la qualité de la connexion
  • Droits d'accès : l'outil accède aux fichiers avec les droits de l'utilisateur connecté. Les sous-dossiers restreints sont silencieusement ignorés
  • Chemin affiché : le chemin réseau complet (Z:\... ou \\serveur\partage\...) est utilisé pour le calcul, ce qui reflète fidèlement ce que OneDrive devra synchroniser

Résultats et export

Les résultats sont exportables au format CSV, directement lisible dans Excel. Chaque ligne contient :

  • Le chemin complet
  • La longueur calculée
  • Les excédents par rapport aux seuils 255 et 400 caractères
  • Le segment le plus long du chemin

Ce fichier peut être partagé avec votre équipe ou intégré dans un rapport de migration.

L'interface distingue clairement les deux niveaux via un code couleur — orange pour les dépassements 255 caractères, rouge pour les dépassements 400 caractères.

Le fichier CSV exporté contient également les deux colonnes d'excédent, ce qui permet de filtrer et trier facilement dans Excel selon le niveau de criticité.

Oui. Chaque ligne affiche :

  • Le chemin complet (ou l'URL SharePoint simulée selon le mode)
  • La longueur calculée
  • Les excédents par rapport aux deux seuils
  • Le segment le plus long du chemin
  • Un bouton d'accès direct au dossier parent dans l'Explorateur Windows
